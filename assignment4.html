Challenge 5: Accessibility Statement

Accessibility concerns of this assignment:
  • In challenge 2, we are building a chessboard using a table and filling it with unicode-equivalents (HTML decimal) of chesspieces. In general, we only want to use tables for tabular data - not for pictures. This is because a screen readers would not know how to interpret this, as many squares are empty. Despite this, the screen reader would read each table and row, and the user would be unlikely to make sense of it or visualize the chessboard as I have designed. Another issue is that colors in this chessboard have specific meaning (in regards to which chesspieces can move where) and the screen reader might not read aloud the colors of the squares. As far as mitigating this - I'm not really sure. I believe this challenge was to practice our styling skills and not actually meant to use the chess board for playing.
  • For challenge 3 I created a form. I learned that using proper semantics and labels is super important here - so the browser knows how to interpret and organize the information. Specifically, this is the first time I've heard of fieldset and legend tags, which are used to group related elements in the form and label them appropriately. This clarifies to screenreaders what the sections are and in what order they should go through sections of the form.
  • Challenge 4 was designed to have students use in-line or internal style tags to mock up a section of information the way it appears in a particular Google Search, as illustrated. Surprisingly, there were two particular issues I noticed with this Google search page, which were directly in conflict with HTML best practices. By inspecting Google's source code, I could see that the subset of links under the first result was organized using a table. This table does not consist of tabular data, so in this instance Google is using the table to align the search results. A good practice of HTML tables is to ONLY use tables to present information that needs to be presented in a table format. Secondly, many of the links were colored (to signify they were links and whether they have been visited or not), but they DO NOT have any other signifying characteristics to indicate they are links. Because some users cannot distinguish colors, it is never advised to use color alone to designate which text is hyperlinked, as explained by W3Schools in their accessibiity guidelines.
